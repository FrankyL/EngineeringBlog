ESP32-based washing machine sensor

Franky Laurentis June 8, 2025

Relatively simple sensor

Aim is to create an IoT sensor that can measure the vibration and temperature of a washing machine.

So roughly the idea for the overall circuit is: ESP32 read piezo for vibration and a thermistor with potential divider for the temperature. Data transmission can really go any way, Thingspeak, MQTT, ESPNOW. For now, I winged it with Thingspeak since I can just reuse my old account with most of the setups done already, just need to copy the API code.

Since Wi-Fi usage is confirmed, can only use ADC1, need to ask ChatGPT to look up the pins for it and since 34-39 is input only and just in case I want to use the other pins for other stuff, I rolled with it.

Quite straightforward, the ESP32 interface with piezoelectric module which is then connected to piezoelectric ceramic plates (not sure if they are actually called this), the potential divider also same thing, standard fare

Pendulum like setup for the double piezo plates, holes at the lower plate allow me to screw in bolts so that it is heavier and amplify the vibration even more with inertia

I also placed foam around the ESP32 to protect the module against excessive vibration

Only downside I regret is the free account on Thingspeak, so 20s per datapoint. I thought would be nice we can go like 5s per datapoint, oh well.

Learnt how to use piezoelectric sensor from this project, or at least the circular flat ceramic plate types. At first I thought it measures the force being exerted on it like a drum, so I have a quick setup to do that but turns out it's not, it creates a signal upon flexing, that is what it perceived when i pressed it with 2 fingers, thus the new pendulum setup.
