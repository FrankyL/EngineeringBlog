Split ortho-linear BLE Keyboard

Franky Laurentis November 25, 2024

Made a conscious option with selecting ortho-linear because it seemed easier, but I hated it after assembling everything.

10/10 don't recommend.

Could have used JLCPCB to assemble one side of the components, but they ended up quite expensive since they calculated with (price per unit * number of buttons) if I remember correctly.

1st set of keyboards assembled with 4xAAA batteries but ended up quite hard to replace as there is no battery cover. Need to remove the whole top side of the casing to replace batteries. Had to do some substitution for the 3rd party keycaps since I forgot to take into account typical keycaps sizing.

2nd set below is also made with 4x Lipo AAA batteries on each side, but printed the casing with battery cover and also tried 3D printing thinner keycaps instead and it doesn't work out all that well with some of them loosening after a few detachments, need to retry with tighter tolerances.

Encountered the same issue with Macropad but on a smaller scale, the numbers also function as F1-F12 when the F button is pressed on either side, and media controls when Media is pressed on either side. Forgot which button does which media function.

Planned to make a later variant that is more conventional, i.e. not split and staggered keys and figure out how to code the firmware to be able to switch between USB and BLE, this one can only do BLE.
