The Making of a Modular 3D-printed Line Following Robot Kit [V1-in progress-untested]

Franky Laurentis June 10, 2025

Introduction and materials

Yes, a kit. I want something that I can sell to people and then they can build it themselves. Maybe also something I can use in the future for my own biz or something.

Of course, you can guess it, *sigh* ESP32, I just bought in bulk a really long time ago and still have like 10 of them unused in my stash alright...I want to start using ESP32-C3 or the S3 but It feels like a waste to not use the ESP32-WROOMs y'know. Honestly it's so insanely cheap it's kinda hard to move on to the newer stuff.

And yeah...L298N, usual fare, we just need 1 for this one. 1 L298N to control 2 motors, 9-12V. The newer stuff however, instead of using the 5 in a line tracer module TCRT5000, we are going to use the single ones, the one we can adjust ourselves or read the analog output instead. I have used the 5 in a line before and the output for the line tracing is finicky at best and over-relies on us adjusting the distance between the module and the terrain.

So the overall setup:

  1. ESP32-WROOM

  2. L298N

  3. Geared motors * 2

  4. Battery, AAA * 6 = 9V total

  5. Color Sensor TCS3200

  6. TCRT5000 module * 3

  7. Ultrasonic distance sensor

To be honest this is actually done with the AI Maker NRC competition as an inspiration, I am aware the inclusion on ultrasonic distance sensor and color sensor is unconventional for a line following robot. Long story about this one (why specifically AI Maker NRC), will tell you guys another day, also skipping the Computer vision part, maybe for V2 or V1_EX

I am also aiming for it to be modular, so modular that you can 3D print new parts adjusted for different rules and different competitions. Maybe that is for V2, for V1 like this, maybe an educational kit is sufficient, made to teach assembly, 3D modelling + printing and soldering. Might consider custom PCB for V2 also.

Extra holes on the base plate / body to allow for replaceable modules / parts. Currently only configurable to 1 form, but I designed it to allow for more reconfigurability in the future. Even right now, we can already easily swap between the 3 * separate TCRT5000 modules and 5-in-a-line TCRT5000 modules. Perhaps even tank treads instead of wheels is possible in near future.

I actually cobbled this one from a previous line following robot but that older one is originally intended to become a sushi delivery robot but I quickly figured that Line Following Robot + FnB is prolly a really bad idea.

Maybe a rail type robot with NEMA17 that does dead reckoning is better for FnB table delivery robots. IIRC, last time I tried, line following + QR code scanning to detect "stations" is iffy sometimes and the reaction time is not that good (I am using Orange Pi Zero, Orange Pi 3 LTS should be reactive enough but also consumes more power). Although also obvious setting up the "rail" in a cafe might costs way more than just using laminated printed papers. Oh well, future me will have to deal with them when they comes.

Older schematic here (Note the usage of 5-in-a-row TCRT5000 module with L1 and L2), I think I will probably retain most of the existing connections even though for this current prototype, the PCB will be hand soldered. As for why there is 2 L1 and L2, I thought I want to make something like microbit's line following robot, where the PCB itself becomes the main body

Assembly is slightly more complicated for the forward section but still OK (hey I planned this one out properly, there's one time I accidentally created a woodworking-esque interlocking mechanism cuz I was too lazy to plan ahead), 3 segments each for 3*TCRT modules, ultrasonic distance sensor and color sensor

I bought this pulley caster wheel set to act as the omni-wheel haha. Don't knock it till you try it, it's pretty good (cheap too).

What's left of prototype sushi delivery robot that I partially copied for the new Line Following Robot :^( Sadge and RIP. Also if you noticed, this is a subtle one, but the new robot has most of the screws facing upwards while this one is more mixed. Obvious issue when using screws and nuts: the nuts have the tendency to fall off when there's movements and vibrations, thus I flipped all the screws to be upwards for the new one. (Also easier to find the bigger screws on the floor haha)

Partial Disassembly/Zoom-ins

Wheel assembly consists of 5 parts for each side including motor, need to buy the 10mm bearing online, quite cheap, the rubber covers for the wheels are optional. The biggest issue would actually be that part you see connected to the motor shaft, you don't want the part to be loose enough to slide easily in cuz the forces from motor would quickly enlarge the socket. However, too small a socket and it's hell to force it in, so need to strike a precise balance.

As some of you might have noticed, the battery holder is cobbled together with 2 4*AAA battery holders, 1 platform bolted above the PCBs and copious amount of hot glue. Why are they facing the same direction? Cuz I'm using battery spring contacts (You can also buy this online). A bit of a pain to solder later on, but works really well in the context of 3D prints, also took multiple tries to get the tolerances right.

Assembly steps to assemble main body, first need to bolt on the PCB platform then bolt on the additional battery platform, a bit of a messy workaround here cuz the base that houses the motor is derived from the earlier sushi robot

Currently this is the progress, testing firmware confirmed all module functionality, still need to test actual Line Following performance, will update this page when there's more results
