[UPDATED] ESP32 Based Mecanum Omniwheel Robot + BLE Android RC App

Franky Laurentis June 20, 2025

My first exposure to the concept of this Mecanum omniwheel is during the VEX robotics competition back in JC, year 1 (2013, damn that's like 12 years ago). I honestly thought the seniors who bought the Mecanum wheels made the wrong purchase (still thought it's the wrong purchase, we definitely needed servos and stronger motors, not fancy omniwheels). We bombed the competition haha but in turn I got some experience sawing through the aluminum frames which so it was pretty fun all in all and playing with Mecanum wheeled robots.

Relatively simple on the electronics side, I originally considered using NRF24 for transmission but ended up deciding to try using purely WiFi, maybe also an excuse to force myself to make Android app for control.

[UPDATE] Made a quick Kotlin BLE Android App originally for the tank tread robot (here). The app is tested on the modular robot kit in a YT vid within the tank tread post. Unlike the tank tread robot which uses 2 seekbars for basic forward/reverse + turning, this mecanum robot will require full 2-axis joystick control for omnidirectional movement, so I will most likely just tack on a new mode to the app.

Straight forward circuit with ESP32, but needed 2 L298N modules since all wheels are independent, 1 L298N support 2 directions, I need independent 4 wheels. There are an onboard 2S LiPo battery (3.7V * 2 = 7.4V nominal, highest at 8.4V, lowest at 6V (not recommended to let it fall this low)) Also, I don't feel like designing a whole ass casing for the battery so I just gave it a platform, put 2 slots and cable tie the thing.

Basic movement for now, implemented using WiFi AP, I'm kind of curious how the direction fusion is implemented actually. The body is planned to be able to change shape (The right side move left, the left side move right, thus folding the body and the middle segment rises upwards), haven't got time to do it yet, but I provisioned a lot of modularity to do it on a later time when I am bored.

If I have a gripe with the wheels (cheap, ordered online), is that I would have liked it to have a hole on both sides, so that it is more stable, right now the weight of the middle segment bent the wheel a bit, it's like on a slight angle when its on the floor moving.

Also HTML based control interface is horrible, need to figure out how to use the BLE Android app with this one.

YT vid of the basic movement, this is done using the WiFi AP and legit only have the gamepad up-down-left-right buttons on an HTML page.

[Update] x-y controls and direction fusion using the basic 2 seekbar controls that I originally made for the tank tread robot

YT Vid here on Mecanum Omniwheel controls using the BLE Remote Controller App. I know I said above that I will tack on a new mode on top of the original tank controls / conventional controls, but I wanna do a quick experiment with the Mecanum direction fusion (x and y only, omega rotation control is omitted for this one) which worked wonderfully even if there's a small omega rotation drift which is expected cuz the build is pretty janky on the wheel side.

If you have never heard direction fusion prior to this, Mecanum omniwheels strongest feature is that the rotation of each wheel is able to determine sideways movement (x-axis) / forward backward (y-axis) and on the spot rotation (omega).
